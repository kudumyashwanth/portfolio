01

Aurora v1

RISC-V + AI ACCELERATOR SoC · RTL-to-GDSII

A multi-clock System-on-Chip pairing a Rocket RV64IMAC CPU tile with a 4×4 systolic int16 matrix-multiply accelerator over an 8×8 AXI4 crossbar — carried through the complete flow on Sky130. Multi-corner timing closure (WNS/TNS = 0), clean DRC, device-perfect LVS (137,562 = 137,562 devices) on a 41.25 mm² die.

  • SystemVerilog
  • AMBA AXI4
  • Async-FIFO CDC
  • OpenLane/OpenROAD
  • Sky130
02

Neural Accelerator

CNN INFERENCE ASIC · AXI4-STREAM

A modular CNN inference accelerator — preprocessing, a sequential MAC convolution engine, ReLU, pooling, and a fully-connected layer — through full RTL-to-GDSII. 32-bit AXI4-Stream with backpressure-aware stalling. 100 MHz target (1.08 ns critical path), WNS/TNS = 0, 4.0 mm² die, 0 DRC, LVS clean.

  • SystemVerilog
  • AXI4-Stream
  • Cadence Xcelium
  • Sequential MAC
  • Sky130
03

The Thousand Sunny

AUTONOMOUS EXPLORATION ROVER · ROS 2

An AI-driven 6WD rover built around a Jetson Orin NX. ROS 2 Humble navigation with SLAM (RTAB-Map, Cartographer) and IMU + LiDAR + RGBD + EKF sensor fusion. YOLOv8 and DeepLabV3+ optimized via TensorRT for real-time inference, validated in NVIDIA Isaac Sim, with a Flask telemetry dashboard over LoRa + Wi-Fi.

  • ROS 2 Humble
  • Jetson Orin NX
  • SLAM
  • TensorRT
  • Isaac Sim
04

Pocket Assistant

VOICE-CONTROLLED AI · ESP32 EMBEDDED

A portable voice assistant on an ESP32 with an I²S microphone and audio amplifier. Interrupt-driven firmware for button-triggered capture, buffered Wi-Fi API calls for AI responses, and a real-time capture→process→playback pipeline. Reworked an open-source PCB for power delivery, signal integrity, and standalone recovery.

  • ESP32 (C/C++)
  • I²S Audio
  • Interrupt FW
  • PCB Design
  • Wi-Fi API
05

AHB → APB Bridge

AMBA 4 PROTOCOL BRIDGE · VERIFICATION

FSM-based protocol translation between a high-performance AHB bus and a low-speed APB bus, with wait-state handling and correct HADDR / HWRITE / HREADY / PSEL / PENABLE / PREADY sequencing. Verified with a constrained-random, assertion-driven testbench — full functional coverage, zero handshake violations.

  • SystemVerilog
  • AMBA 4
  • UVM-style TB
  • Functional Coverage
  • Assertions
